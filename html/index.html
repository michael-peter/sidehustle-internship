This is a paragraph with a table after it. HTML tables have two basic sections, the header (first row with titles/labels) and other rows. The table is declared/initiated with a "table" tag. Inside the table tag, we have table rows, with "tr" tags. Each row has data in it, with the "td" tag. However, the first row, if we want it so, could be the labels/column-headers and we use the "th" to signify this. "th" means table header. So we use "th" for the first row and "td" for others (if we want a table header). In more critical implementation, though, we would have the first row or all rows that mark the table head under the "thead" tag and all other rows under the "tbody"

We use line breaks ubiquitously because all elememts in a form tag are displayed inline. "br" allows me to render them in new lines

The form inputs can have default values and these are represented by the attribute "value". The "submit" input type is a button which submits the form [programmatically] while the "reset" input type resets the inputs to the initial values we put or empties them if we didn't put initial values. The "value" attribute is also used for the submit and reset buttons so as to label them (since they are buttons and not text fields). There are other attributes and one of them is "placeholder" which only labels a field but is not an actual text that one can delete (it is replaced as soon as the user types into the field). It is used to hint the user what information is expected in that field.
